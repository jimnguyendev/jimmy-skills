## Outcomes Pack

4 skills for outcome thinking, roadmap reframing, operating model design, and transformation by behavior change.

### Routing

- Use `outcome-thinking` when the user needs to separate outputs, outcomes, and impact, or define leading indicators and behavioral OKRs.
- Use `outcomes-based-planning` when roadmap items need to become hypotheses, experiments, or journey-based plans.
- Use `organizing-for-outcomes` when team structure, prioritization, intake, or stakeholder trust need to shift away from feature silos.
- Use `outcomes-driven-transformation` when the target is internal adoption, new habits, or organizational behavior change.

### Skills

| Skill | When to use |
|---|---|
| `outcome-thinking` | Defines outcomes as behavioral change, separates outcomes from outputs and impacts, and writes better outcome-based OKRs. |
| `outcomes-based-planning` | Turns roadmap items into hypotheses, journey-based plans, and measurable experiments. |
| `organizing-for-outcomes` | Team-structure and operating-model guidance for cross-functional work organized around outcomes instead of component silos. |
| `outcomes-driven-transformation` | Guides internal change programs where the target is stakeholder adoption, new habits, and organizational behavior change. |

### Relationship to Other Packs

- Data platform and analytics implementation → `packs/data-engineer/`
- General engineering process → `packs/engineering/`
- This pack provides the outcome-design and operating-model layer that can sit above product, platform, or transformation work.
